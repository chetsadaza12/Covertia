# Command Workflow

ตัวอย่าง Workflow สาธิตการทำงานของบล็อก **Command** ที่ใช้สำหรับการรันคำสั่งระบบผ่านช่องทางที่กำหนด (Cmd, Terminal, หรือ PowerShell) เหมาะสำหรับควบคุมโปรแกรมภายนอกหรือดึงผลลัพธ์การรันจากระบบปฏิบัติการโดยตรง โดยเชื่อมต่อจากบล็อก **Start (Trigger)** ไปยังบล็อก **Open URL** และตามด้วยบล็อก **Command**

## Workflow Blocks

| Block | Type | Description |
|-------|------|-------------|
| **Start (Trigger)** | `BlockBasic` | จุดเริ่มต้นการทำงาน พร้อมระบบ Parameters สำหรับสร้าง UI Form รับข้อมูลผู้ใช้ |
| **Open URL** | `BlockBasicWithFallback` | เปิดหน้าเว็บเป้าหมาย (ในตัวอย่างใช้ `facebook.com`) |
| **Command** | `BlockBasicWithFallback` | รันชุดคำสั่งบนระบบปฏิบัติการตามช่องทางที่เลือก (Cmd/Terminal/PowerShell) และบันทึกผลลัพธ์ได้ |

## การตั้งค่าบล็อก Command

บล็อก **Command** เป็นโหมดขั้นสูงที่ช่วยให้สามารถเลือกช่องทางการรันคำสั่ง (Command Executor) บนระบบของเครื่องได้หลากหลายรูปแบบตามประเภทของระบบปฏิบัติการ

### ค่าพารามิเตอร์

| พารามิเตอร์ | คำอธิบาย | ตัวเลือก / รายละเอียดเพิ่มเติม |
| :--- | :--- | :--- |
| **Command Executor** | ช่องทางหรือสภาพแวดล้อมที่ใช้ประมวลผลคำสั่ง | `Cmd` (Command Prompt บน Windows) <br/> `Terminal` (Bash/Zsh บน macOS/Linux) <br/> `PowerShell` (บน Windows) |
| **Command** | ชุดคำสั่งที่ต้องการรัน | รองรับการใส่ข้อความตรง ๆ หรือแทรกตัวแปร `{{variables}}` |
| **Assign to variable** | บันทึกข้อความการทำงานตอบกลับ (stdout) ไปเก็บลงในตัวแปร | เปิด/ปิด (Boolean) |
| **Variable name** | กำหนดชื่อตัวแปรเมื่อต้องการจัดเก็บผลลัพธ์ | ตัวอย่างเช่น: `shell_output` |
| **Regular expression (Regex)** | รูปแบบ Regex เพื่อดึงเฉพาะผลลัพธ์ส่วนที่ต้องการ | ตัวอย่างเช่น: `[a-zA-Z]+` เพื่อเอาเฉพาะข้อความภาษาอังกฤษ |
| **Timeout** | ระยะเวลาจำกัดสูงสุดในการรันคำสั่ง (มิลลิวินาที) | ป้องกันไม่ให้ตัวรันสคริปต์ค้างหากคำสั่งทำงานไม่สิ้นสุด |
| **Delay time** | เวลาหน่วงก่อนเริ่มดำเนินงาน (มิลลิวินาที) | หน่วยเป็นมิลลิวินาที (เช่น `1000` ms = 1 วินาที) |

### กรณีการนำไปใช้งาน (Use Cases)

*   **รันสคริปต์ PowerShell ขั้นสูง**: สำหรับเครื่อง Windows สามารถใช้โหมด PowerShell รันสคริปต์เพื่อจัดแจงสิทธิ์ เข้าถึงข้อมูลระบบ หรือดาวน์โหลดไฟล์ที่ระบบปกติบล็อกไว้
*   **ใช้งานข้ามแพลตฟอร์ม (Cross-platform Terminal)**: เลือกใช้โหมด `Terminal` เพื่อรันคำสั่งสคริปต์ Bash/Shell บนระบบ macOS หรือ Linux
*   **จัดการข้อมูลเบื้องหลัง**: สั่งรันชุดคำสั่ง Cmd เพื่อสร้างโฟลเดอร์ คัดลอกเอกสาร หรือส่งคำสั่งพิมพ์ไปยังเครื่องพิมพ์เครือข่าย

---

### ลำดับการทำงานของ Workflow นี้

1.  **Start (Trigger)** — เริ่มทำงานและรันพารามิเตอร์ของระบบเริ่มต้น
2.  **Open URL** — เปิดหน้าเบราว์เซอร์ไปที่เป้าหมาย `https://www.facebook.com/`
3.  **Command** — ดำเนินการรันคำสั่งระบบผ่านช่องทางที่เลือก (ในตัวอย่างเริ่มต้นใช้ `Cmd`) และเก็บค่าข้อมูลตอบกลับ
