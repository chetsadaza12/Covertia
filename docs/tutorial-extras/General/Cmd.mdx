# Cmd Workflow

ตัวอย่าง Workflow สาธิตการทำงานของบล็อก **Cmd** ที่ใช้สำหรับการรันคำสั่งเชลล์ (Shell command) หรือสคริปต์คำสั่งของระบบปฏิบัติการเพื่อควบคุมหรือเรียกโปรแกรมภายนอก โดยเชื่อมต่อจากบล็อก **Start (Trigger)** ไปยังบล็อก **Open URL** และส่งต่อข้อมูลไปยังบล็อก **Cmd**

## Workflow Blocks

| Block | Type | Description |
|-------|------|-------------|
| **Start (Trigger)** | `BlockBasic` | จุดเริ่มต้นการทำงาน พร้อมระบบ Parameters สำหรับสร้าง UI Form รับข้อมูลผู้ใช้ |
| **Open URL** | `BlockBasicWithFallback` | เปิดหน้าเว็บเป้าหมาย (ในตัวอย่างใช้ `facebook.com`) |
| **Cmd** | `BlockBasicWithFallback` | รันชุดคำสั่งระบบ (System Command / Shell Script) พร้อมดึงผลลัพธ์การรันคำสั่งกลับมาใช้งานได้ |

## การตั้งค่าบล็อก Cmd

บล็อก **Cmd** สามารถใช้สั่งงานระบบคอมพิวเตอร์ผ่าน Command Line และเก็บค่าผลลัพธ์การพิมพ์ตอบกลับ (stdout) ไปเก็บลงในตัวแปรระบบตามเงื่อนไขที่กำหนด

### ค่าพารามิเตอร์

| พารามิเตอร์ | คำอธิบาย | รายละเอียดเพิ่มเติม |
| :--- | :--- | :--- |
| **Command** | ชุดคำสั่งคอมพิวเตอร์ที่ต้องการรัน | เช่น `echo "Hello Covertia"`, คำสั่ง Node.js, Python หรือคำสั่ง CLI อื่น ๆ |
| **Assign to variable** | เปิดใช้งานเพื่อนำผลลัพธ์ที่ได้จากการรันสคริปต์ไปเซฟลงตัวแปร | เปิด/ปิด (Boolean) |
| **Variable name** | ตั้งชื่อตัวแปรที่ใช้จัดเก็บข้อมูล | ตัวอย่างเช่น: `cli_output` |
| **Regular expression (Regex)** | ข้อกำหนด Regex สำหรับดึงเฉพาะส่วนที่ต้องการจากข้อความตอบกลับ | ตัวอย่างเช่น: `\d+` สำหรับดึงเฉพาะเลขที่ได้จาก CLI |
| **Timeout** | ระยะเวลาจำกัดในการทำงานของบล็อก (มิลลิวินาที) | หน่วยเป็นมิลลิวินาที (เช่น `5000` ms) |
| **Delay time** | เวลาหน่วงก่อนที่บล็อกนี้จะเริ่มดำเนินการทำงาน (มิลลิวินาที) | หน่วยเป็นมิลลิวินาที (เช่น `1000` ms = 1 วินาที) |

### กรณีการนำไปใช้งาน (Use Cases)

*   **รันสคริปต์งานภายนอก**: เรียกใช้งาน Python หรือ Batch script เพื่อจัดการเตรียมข้อมูลประเภท Excel / CSV ก่อนจะเริ่มเข้าเว็บเบราว์เซอร์
*   **ตรวจสอบสถานะเน็ตเวิร์ก**: ใช้รันคำสั่ง `ping` หรือ `curl` ภายนอกเพื่อดูความเสถียรก่อนเข้าทำงาน
*   **สกัดข้อมูลของเครื่อง**: ดึงชื่อเครื่อง (Hostname) หรือที่อยู่ IP ของผู้ใช้งานเครื่องมารวบรวมสรุปในสเปรดชีต
*   **สั่งงานระบบ (System Control)**: เช่น รันคำสั่งเปิดเครื่องพิมพ์ สั่งลบโฟลเดอร์ขยะ หรือเปิดแอปพลิเคชันอื่นของเครื่อง

---

### ลำดับการทำงานของ Workflow นี้

1.  **Start (Trigger)** — เริ่มทำงานและโหลด UI พารามิเตอร์เริ่มต้น
2.  **Open URL** — เปิดหน้าเบราว์เซอร์ไปที่เป้าหมาย `https://www.facebook.com/`
3.  **Cmd** — เริ่มรันสคริปต์ตามคำสั่งและส่งคืนข้อมูลผลลัพธ์กลับมาเก็บในตัวแปร
