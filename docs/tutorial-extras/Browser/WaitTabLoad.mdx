# Wait Tab Load

**Wait Tab Load** คือบล็อกคำสั่งสำหรับสั่งให้เวิร์กโฟลว์ **หยุดรอจนกว่าแท็บหน้าเว็บเบราว์เซอร์ปัจจุบันจะโหลดเสร็จสิ้นสมบูรณ์** ก่อนที่จะข้ามไปทำงานในบล็อกถัดไป

คำสั่งนี้มีความสำคัญอย่างยิ่งในการทำ Web Automation เนื่องจากอินเทอร์เน็ตหรือหน้าเซิร์ฟเวอร์ปลายทางแต่ละแห่งมีความเร็วในการโหลดหน้าเว็บไม่เท่ากัน การเพิ่มบล็อกนี้จะช่วยป้องกันปัญหาบอตรันสคริปต์เร็วเกินไปจนหาปุ่มหรือฟิลด์ข้อมูลบนหน้าจอไม่พบ (Element Not Found)

---

## รูปแบบสถานะการรอโหลด (Navigation States)

โดยทั่วไปบล็อกประเภทนี้จะเปิดโอกาสให้เลือกเงื่อนไขการโหลดเสร็จของเว็บเพจ ดังนี้:

1.  **Load**: รอจนกระทั่งเหตุการณ์ `onload` ของหน้าเว็บทำงาน (โหลดโค้ด HTML, CSS, JavaScript และรูปภาพทั้งหมดเสร็จสิ้น)
2.  **DOMContentLoaded**: รอเพียงแค่โครงสร้างเอกสาร HTML หลักโหลดเสร็จและพร้อมประมวลผล (ไม่ต้องรอให้ไฟล์รูปภาพหรือ CSS ขนาดใหญ่โหลดเสร็จทั้งหมด) ช่วยให้บอตรันงานต่อได้รวดเร็วยิ่งขึ้น
3.  **Network Idle**: รอจนกว่าจะไม่มีคำขอรับส่งข้อมูลทางเครือข่ายเพิ่มเติม (ไม่มีการดาวน์โหลดข้อมูลใด ๆ ผ่าน Network เพิ่มเติม) มักนำมาใช้กับเว็บประเภท Single Page Application (SPA) ที่ใช้ AJAX ดึงข้อมูลมาแสดงผลต่อเนื่อง

## พารามิเตอร์การตั้งค่าหลัก

*   **Timeout (millisecond)**: ระยะเวลาสูงสุดที่อนุญาตให้ระบบรอ หากหน้าเว็บโหลดช้าเกินกว่าเวลาที่กำหนดนี้ (เช่น เกิน 30 วินาที) ระบบจะตัดการทำงาน (Timeout) หรือไปทำเงื่อนไขถัดไปเพื่อป้องกันไม่ให้สคริปต์ค้าง
*   **Delay time (millisecond)**: เวลาหน่วงเพิ่มเติมหลังจากระบบตรวจพบว่าหน้าเว็บโหลดเสร็จแล้ว เพื่อความมั่นใจว่าระบบพร้อมเต็มที่ก่อนเริ่มคลิกหรือกรอกข้อมูล

---

## กรณีการใช้งาน (Use Cases)

*   **หลังจากการคลิกลิงก์สำคัญ**: เมื่อใช้บล็อก Mouse Click เพื่อกดปุ่มล็อกอินหรือปุ่มเปลี่ยนหน้าเว็บเพจ ควรต่อท้ายด้วย **Wait Tab Load** ทันที เพื่อรอให้หน้าเว็บถัดไปแสดงผลเสร็จเรียบร้อยก่อนที่บอตจะพิมพ์รหัสหรือคลิกทำรายการต่อ
*   **ป้องกันการรันสคริปต์ล้มเหลว (Robustness)**: ป้องกันไม่ให้ขั้นตอนถัดไปล้มเหลวเนื่องจากการโหลดรูปภาพหรือ JavaScript ของหน้าเว็บดีเลย์
