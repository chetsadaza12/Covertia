# Network Toggle Workflow

ตัวอย่าง Workflow สาธิตการทำงานของบล็อก **Network Toggle** ซึ่งเป็นคำสั่งสำหรับการเปิดหรือปิดการเชื่อมต่ออินเทอร์เน็ตของแท็บหน้าต่างเบราว์เซอร์เป้าหมาย (Network Connection Switch) เพื่อใช้ในการทดสอบโหมด Offline หรือกรณีที่ต้องการหยุดโหลดข้อมูลเว็บเพจระหว่างรันงานเพื่อประหยัดทรัพยากร

## Workflow Blocks

| Block | Type | Description |
|-------|------|-------------|
| **Start (Trigger)** | `BlockBasic` | จุดเริ่มต้นทำงานเวิร์กโฟลว์ |
| **Open URL** | `BlockBasicWithFallback` | เปิดหน้าเบราว์เซอร์ไปยัง URL ที่ต้องการ |
| **Network Toggle** | `BlockBasicWithFallback` | สลับสถานะเปิด/ปิดการเชื่อมต่อเครือข่ายอินเทอร์เน็ตของแท็บเป้าหมาย |

## การตั้งค่าบล็อก Network Toggle

บล็อก **Network Toggle** ช่วยให้คุณจำลองสถานะการเชื่อมต่อ Network ในระดับแท็บ

### ค่าพารามิเตอร์หลัก

| พารามิเตอร์ | คำอธิบาย | ตัวเลือก / รายละเอียดเพิ่มเติม |
| :--- | :--- | :--- |
| **Description** | คำอธิบายรายละเอียดสั้น ๆ สำหรับบล็อกนี้ | แสดงผลเป็นฟิลด์หัวเรื่องไร้กรอบด้านบนสุด |
| **Scope** | ขอบเขตการเปิด/ปิดเน็ตของเบราว์เซอร์ | • `Active tab` (เฉพาะแท็บที่กำลังทำงานอยู่)<br/>• `All tabs` (ทุกแท็บในเบราว์เซอร์) |
| **Status Switch**| สถานะการเชื่อมต่ออินเทอร์เน็ตที่ต้องการสลับเปลี่ยน | • `Close` (ปิดเน็ต / ตัดการเชื่อมต่อ / Offline)<br/>• `Open` (เปิดเน็ต / เชื่อมต่อปกติ / Online) |

### ค่าในส่วน Settings

*   **Timeout (millisecond)**: กำหนดเวลาสูงสุดในการทำงานของบล็อกนี้
*   **Delay time (millisecond)**: เวลาหน่วงก่อนเริ่มประมวลผลคำสั่งในบล็อก

---

### กรณีการใช้งาน (Use Cases)

*   **Offline Mode Testing**: จำลองเหตุการณ์เน็ตหลุดหรือไม่มีสัญญาณอินเทอร์เน็ต (Offline) เพื่อตรวจสอบว่าเว็บแอปพลิเคชันหรือตัวสคริปต์อัตโนมัติจะจัดการและตอบสนองอย่างไร (เช่น มีการแจ้งเตือนหรือแคชข้อมูลไว้หรือไม่)
*   **Asset Loading Block / Performance**: สั่งปิดการเชื่อมต่อ (`Close`) ทันทีหลังจากเปิดหน้าเว็บเพจสำเร็จ เพื่อป้องกันไม่ให้เว็บเพจดึงข้อมูลสคริปต์ โฆษณา หรือรูปภาพเพิ่มเติมอื่น ๆ ที่อาจทำให้บอตรันช้าลง ช่วยประหยัดความเร็วและทรัพยากรเครื่อง
