# Release Ram

**Release Ram** คือบล็อกคำสั่งสำหรับจัดการทรัพยากรหน่วยความจำของหน้าเบราว์เซอร์ (Browser Memory Optimization) ทำหน้าที่ช่วย **คืนพื้นที่หน่วยความจำแรม (RAM)** ที่ถูกใช้งานโดยแท็บต่าง ๆ ที่เปิดค้างไว้ เพื่อช่วยลดการใช้ทรัพยากรเครื่องคอมพิวเตอร์ และป้องกันปัญหาบอตรันงานช้าลงหรือโปรแกรมค้างเนื่องจากแรมเต็มเมื่อต้องทำงานเป็นระยะเวลานาน

---

## พารามิเตอร์การตั้งค่าหลัก

บล็อกนี้มาพร้อมกับตัวเลือกการคืนแรมตามขอบเขตความต้องการของคุณ:

### 1. Scope (ขอบเขตการคืนแรม)
ผู้ใช้งานสามารถเลือกขอบเขตพื้นที่เบราว์เซอร์ที่ต้องการปลดปล่อยแรมได้ 2 รูปแบบหลัก:

*   **Active Page** (หรือ Active Tab): ทำการเคลียร์แรมและจัดสรรทรัพยากรใหม่เฉพาะหน้าเว็บเพจ/แท็บหลักที่กำลังเปิดใช้งานและรันสคริปต์อยู่ ณ ปัจจุบัน
*   **All Page** (หรือ All Tabs): ทำการเคลียร์แรมของหน้าเว็บเพจ/แท็บทั้งหมดที่เปิดขึ้นมาภายในระบบเบราว์เซอร์ รวมถึงแท็บพื้นหลัง (Background Tabs) ที่ไม่ได้ใช้งานในขณะนั้น

---

## กรณีการใช้งาน (Use Cases)

*   **Long-Running Workflows**: สำหรับเวิร์กโฟลว์ที่จำเป็นต้องรันงานติดต่อกันเป็นเวลาหลายชั่วโมง หรือเปิดหน้าเว็บที่มีปริมาณสื่อ (รูปภาพ วิดีโอ หรือสคริปต์) เป็นจำนวนมาก ซึ่งมักประสบปัญหาเบราว์เซอร์กินแรมสะสม การใส่บล็อก **Release Ram** ไว้ระหว่างจุดต่าง ๆ ของสคริปต์จะช่วยรักษาระดับการกินทรัพยากรให้คงที่
*   **Multi-Tab Tasks**: เมื่อเวิร์กโฟลว์จำเป็นต้องเปิดหน้าต่างหลายแท็บพร้อมกันเพื่อดึงข้อมูล การสั่งคืนแรมให้แท็บที่ไม่ได้ใช้บ่อยด้วยคำสั่ง `All Page` จะช่วยให้ระบบโดยรวมรันได้อย่างลื่นไหล
