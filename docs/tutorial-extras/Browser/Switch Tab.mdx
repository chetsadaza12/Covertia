# Switch Tab Workflow

ตัวอย่าง Workflow สาธิตการทำงานของบล็อก **Switch Tab** ซึ่งเป็นคำสั่งสำหรับค้นหาและสลับไปยังแท็บเบราว์เซอร์เป้าหมายที่ต้องการ ช่วยให้เวิร์กโฟลว์สามารถทำงานข้ามแท็บหลายแท็บได้อย่างราบรื่น ไม่ว่าจะเป็นการค้นหาตาม URL Pattern, ชื่อแท็บ, ตำแหน่งลำดับ, หรือเลื่อนไปแท็บถัดไป/ก่อนหน้า

## Workflow Blocks

| Block | Type | Description |
|-------|------|-------------|
| **Start (Trigger)** | `BlockBasic` | จุดเริ่มต้นทำงานเวิร์กโฟลว์ |
| **Open URL** | `BlockBasicWithFallback` | เปิดหน้าเบราว์เซอร์เป้าหมาย |
| **Switch Tab** | `BlockBasicWithFallback` | ค้นหาและสลับไปยังแท็บเบราว์เซอร์ที่ต้องการ |

## การตั้งค่าบล็อก Switch Tab

### ค่าพารามิเตอร์หลัก

| พารามิเตอร์ | คำอธิบาย | ตัวเลือก / รายละเอียดเพิ่มเติม |
| :--- | :--- | :--- |
| **Description** | คำอธิบายรายละเอียดสั้น ๆ สำหรับบล็อกนี้ | ฟิลด์หัวเรื่องไร้กรอบด้านบนสุด |
| **Find tab by** | วิธีการค้นหาแท็บเป้าหมาย | • `Match patterns` – ค้นหาตาม URL pattern<br/>• `Tab title` – ค้นหาตามชื่อแท็บ<br/>• `Next tab` – สลับไปแท็บถัดไป<br/>• `Previous tab` – สลับไปแท็บก่อนหน้า<br/>• `Tab index` – ระบุตำแหน่งลำดับแท็บโดยตรง |
| **Match Patterns** | URL Pattern สำหรับจับคู่ (Glob pattern) | ปรากฏเมื่อเลือก `Match patterns` เช่น `https://example.com/*` รองรับตัวแปร `{x}` |
| **Tab title** | ชื่อแท็บที่ต้องการค้นหา | ปรากฏเมื่อเลือก `Tab title` รองรับตัวแปร `{x}` |
| **Tab index** | ลำดับตำแหน่งของแท็บ (เริ่มจาก 0) | ปรากฏเมื่อเลือก `Tab index` |
| **Create if there's no match** | สร้างแท็บใหม่หากค้นหาไม่พบ | Checkbox ปรากฏเมื่อเลือก `Match patterns` หรือ `Tab title` |
| **Set as active tab** | ตั้งแท็บเป้าหมายเป็นแท็บที่กำลังใช้งาน (Focus) | Checkbox สีน้ำเงิน เปิดเป็นค่าเริ่มต้น |

### ค่าในส่วน Settings

*   **Timeout (millisecond)**: กำหนดเวลาสูงสุดในการค้นหาแท็บเป้าหมาย
*   **Delay time (millisecond)**: เวลาหน่วงก่อนเริ่มประมวลผลคำสั่ง

---

### กรณีการใช้งาน (Use Cases)

*   **Multi-Tab Data Collection**: เปิดหลายแท็บพร้อมกัน (เช่น ดึงข้อมูลจากหลายหน้าเว็บ) แล้วใช้ Switch Tab ค้นหาตาม URL Pattern เพื่อสลับไปยังแท็บที่ต้องการประมวลผลข้อมูลต่อ
*   **Tab Navigation Loop**: ใช้โหมด `Next tab` หรือ `Previous tab` ร่วมกับ Loop เพื่อวนทำงานข้ามแท็บทีละตัวอัตโนมัติ เช่น เก็บข้อมูลจากทุกแท็บที่เปิดอยู่
