# Delay Workflow

ตัวอย่าง Workflow สาธิตการทำงานของบล็อก **Delay** ที่ใช้สำหรับหยุดรอ (หน่วง) การทำงานตามระยะเวลาที่กำหนดก่อนดำเนินการบล็อกถัดไป โดย Workflow นี้เริ่มจากบล็อก **Start (Trigger)** เชื่อมต่อไปยัง **Open URL** เพื่อเปิดหน้าเว็บเป้าหมาย แล้วจึงต่อเข้าบล็อก **Delay** เพื่อหน่วงเวลาก่อนดำเนินการต่อ

## Workflow Blocks

| Block | Type | Description |
|-------|------|-------------|
| **Start (Trigger)** | `BlockBasic` | จุดเริ่มต้นการทำงาน พร้อมระบบ Parameters สำหรับสร้าง UI Form รับข้อมูลผู้ใช้ |
| **Open URL** | `BlockBasicWithFallback` | เปิดหน้าเว็บเป้าหมาย (ในตัวอย่างใช้ `facebook.com`) |
| **Delay** | `BlockDelay` | หน่วงเวลาการทำงานตามจำนวนมิลลิวินาทีที่กำหนดก่อนไปบล็อกถัดไป |

## การตั้งค่าบล็อก Delay

บล็อก **Delay** ใช้สำหรับหยุดรอการทำงานชั่วคราวก่อนที่ Workflow จะดำเนินการบล็อกถัดไป เหมาะสำหรับการเว้นระยะเวลาระหว่างการกระทำ เช่น รอให้หน้าเว็บโหลดเสร็จ หรือป้องกันการถูกตรวจจับจากการทำงานเร็วเกินไป

### ค่าพารามิเตอร์

| พารามิเตอร์ | คำอธิบาย | ค่าเริ่มต้น |
| :--- | :--- | :--- |
| **Time (ms)** | ระยะเวลาที่ต้องการหน่วง หน่วยเป็นมิลลิวินาที (1000 ms = 1 วินาที) | `5000` |

### ตัวอย่างการใช้งาน

| ค่า Time (ms) | ระยะเวลาจริง | กรณีใช้งาน |
| :--- | :--- | :--- |
| `1000` | 1 วินาที | หน่วงเวลาสั้น ๆ ระหว่างการคลิกปุ่ม |
| `3000` | 3 วินาที | รอให้ AJAX Request โหลดเสร็จ |
| `5000` | 5 วินาที | รอให้หน้าเว็บโหลดเนื้อหาทั้งหมด |
| `10000` | 10 วินาที | เว้นระยะนานก่อนดำเนินการถัดไปเพื่อจำลองพฤติกรรมผู้ใช้จริง |

---

### เมื่อไหร่ควรใช้ Delay

*   **รอหน้าเว็บโหลด**: ใช้ Delay หลังบล็อก Open URL เพื่อรอให้หน้าเว็บโหลดเสร็จสมบูรณ์ก่อนดำเนินการต่อ
*   **ป้องกันการถูกตรวจจับ (Anti-detection)**: เว้นระยะเวลาระหว่างการกระทำเพื่อจำลองพฤติกรรมผู้ใช้งานจริง ลดโอกาสที่เว็บไซต์จะตรวจจับว่าเป็นบอท
*   **รอ API Response**: ใช้หน่วงเวลาระหว่างการเรียก API เพื่อไม่ให้เกิด Rate Limit
*   **ควบคุมจังหวะการทำงาน**: ใช้เพื่อจัดจังหวะขั้นตอนต่าง ๆ ของ Workflow ให้ทำงานอย่างราบรื่น

---

### ลำดับการทำงานของ Workflow นี้

1.  **Start (Trigger)** — รับข้อมูลจากผู้ใช้ผ่าน Parameter Form (username, password, proxy config ฯลฯ)
2.  **Open URL** — เปิดหน้าเว็บ `https://www.facebook.com/` ในแท็บใหม่
3.  **Delay** — หน่วงเวลา `5000 ms` (5 วินาที) ก่อนดำเนินการบล็อกถัดไป
