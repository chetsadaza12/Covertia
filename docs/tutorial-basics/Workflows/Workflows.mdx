# Workflows

Workflows allow you to create, import, and manage automated workflow scripts. These scripts can run custom actions and step-by-step browser instructions across your profiles.

---

## Workflows Dashboard

The Workflows dashboard provides a centralized interface for viewing and executing automation workflows.

---

### Dashboard Controls

#### Search
Filter workflows by entering text keywords into the search box to locate specific scripts quickly.

#### New workflow
Opens the creation dialog to add a new workflow to your list.

#### Import
Upload and import externally saved workflow files into your dashboard.

#### Delete
Permanently remove selected workflows from your list.

---

### Workflows Table

The table lists all available automation workflows with the following columns:

#### Checkbox & Index (#)
Allows selecting multiple workflows for bulk deletion. The index shows the row number.

#### Name
Displays the workflow's active version tag and name.
* *Example*: `v1.0.0 {Auto Login} Facebook Www (Set 2)`
* *Example*: `v6.0.4 Facebook KM V6.0.5`

#### Author
Identifies the creator or developer of the automation workflow (e.g., `jacob jacob` or `ヱスジ`).

#### Description
Shows a summary or notes about what the workflow performs.

#### Last Interact
Indicates when the workflow was last edited, updated, or executed (e.g., `12 hours ago` or `5 months ago`).

#### Status & Actions (#)
Displays a lock icon (green lock indicating status) and a three-dot menu button to access specific workflow settings and actions.

---

## Create Workflow

Creating a new workflow opens the **Workflow** configuration dialog.

---

### Configuration Fields

#### Name
Enter a unique identifier for the workflow. It is recommended to include the platform name or target purpose.

#### Description
Add details about the steps, requirements, or instructions for the workflow.
* **Character Counter**: The description input has a limit of **300 characters** (displays progress as `0/300`).

---

### Actions

* **Create**: Save the details and create the new workflow.
* **Cancel**: Discard changes and close the dialog.

---

## Best Practices

```diff id="wfbest1"
+ Write short version numbers in your workflow names to track updates easily.
+ Keep descriptions clear and under 300 characters to briefly guide team members.
```
