# Clipboard Workflow

ตัวอย่าง Workflow สาธิตการทำงานของบล็อก **Clipboard** ที่ใช้สำหรับจัดการข้อมูลในคลิปบอร์ด (คัดลอก/ดึงค่า) เพื่อนำข้อมูลที่ได้ไปใช้งานในบล็อกถัดไป หรือบันทึกลงฐานข้อมูล โดยตัวอย่างนี้ต่อยอดจากบล็อก **Start (Trigger)** ไปยังบล็อก **Open URL** และสิ้นสุดที่บล็อก **Clipboard**

## Workflow Blocks

| Block | Type | Description |
|-------|------|-------------|
| **Start (Trigger)** | `BlockBasic` | จุดเริ่มต้นการทำงาน พร้อมระบบ Parameters สำหรับสร้าง UI Form รับข้อมูลผู้ใช้ |
| **Open URL** | `BlockBasicWithFallback` | เปิดหน้าเว็บเป้าหมาย (ในตัวอย่างใช้ `facebook.com`) |
| **Clipboard** | `BlockBasicWithFallback` | จัดการคัดลอกข้อความ (Set) หรือรับค่าข้อความ (Get) จากคลิปบอร์ดของระบบ |

## การตั้งค่าบล็อก Clipboard

บล็อก **Clipboard** มีการทำงาน 2 โหมดหลัก คือ **Get** (ดึงข้อความจากคลิปบอร์ดมาใช้) และ **Set** (คัดลอกข้อความลงคลิปบอร์ด) โดยปรับตั้งค่าได้ผ่านฟังก์ชันดังต่อไปนี้:

### ค่าพารามิเตอร์

| พารามิเตอร์ | คำอธิบาย | รายละเอียดเพิ่มเติม |
| :--- | :--- | :--- |
| **Type** | เลือกโหมดการทำงานของบล็อก | `get` (อ่านค่า) / `set` (เขียนค่า) |
| **Assign variable** | กำหนดข้อมูลจากคลิปบอร์ดไปยังตัวแปร | ติ๊กเลือกหากต้องการเซฟเก็บไว้เป็นตัวแปรระบบ |
| **Variable name** | ตั้งชื่อตัวแปรรับค่า | ตัวอย่างเช่น: `clipboard_text` |
| **Save data** | บันทึกข้อมูลลงตารางข้อมูลหลักของสคริปต์ | เปิด/ปิด (Boolean) |
| **Data column** | เลือกคอลัมน์ในตารางที่จะจัดเก็บค่านี้ | คอลัมน์ที่ตั้งไว้ในส่วนของ Database |
| **Data to copy** | ข้อความที่ต้องการเซฟลงคลิปบอร์ด (ใช้เฉพาะในโหมด `Set`) | รองรับการใส่ข้อความตรง ๆ หรือใช้ตัวแปร `{{variables}}` |
| **Copy selected text** | คัดลอกเฉพาะตัวอักษรที่กำลังคลิกไฮไลต์อยู่บนเว็บหน้าปัจจุบัน | ทำงานคล้ายการจำลองกด `Ctrl + C` บนส่วนที่เลือก |
| **Delay** | เวลาหน่วงก่อนทำคำสั่งจัดการคลิปบอร์ด (มิลลิวินาที) | หน่วยเป็นมิลลิวินาที (เช่น `500` ms) |

---

### กรณีการนำไปใช้งาน (Use Cases)

*   **โหมด Get (อ่านค่าจากคลิปบอร์ด)**:
    *   ดึงรหัสยืนยันตัวตน (2FA / OTP Code) หรือลิงก์ล็อกอินที่คัดลอกมาจากแหล่งภายนอก เพื่อเตรียมไปกรอกในหน้าเว็บ
    *   รับค่าคีย์ API หรือโทเค็นเพื่อไปรันสคริปต์ขั้นตอนถัดไป
*   **โหมด Set (เขียนค่าลงคลิปบอร์ด)**:
    *   คัดลอกข้อความแจ้งเตือน, คำทักทาย หรือคอมเมนต์สำเร็จรูป เพื่อเอาไปสั่งพิมพ์วาง (Paste) ลงในเบราว์เซอร์เป้าหมายจำลองการพิมพ์จริง
    *   คัดลอกชุดข้อมูลบัญชีเพื่อเอาไปคลิกวางโดยไม่ต้องจำลองพิมพ์ทีละตัว

---

### ลำดับการทำงานของ Workflow นี้

1.  **Start (Trigger)** — เริ่มทำงานและรันพารามิเตอร์ Form (เช่น username, password, proxy config ฯลฯ)
2.  **Open URL** — ดึงข้อมูลเบราว์เซอร์ไปเปิดที่หน้าเว็บเป้าหมาย `https://www.facebook.com/`
3.  **Clipboard** — จัดการดึงข้อมูลจากคลิปบอร์ดมาบันทึกในระบบ (โหมด Get) หรือคัดลอกข้อความออกไปตามฟิลด์ที่กำหนด
