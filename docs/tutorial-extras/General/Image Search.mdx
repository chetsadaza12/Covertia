# Image Search Workflow

ตัวอย่าง Workflow สาธิตการทำงานของบล็อก **Image Search** ที่ใช้สำหรับค้นหารูปภาพเป้าหมายบนหน้าจอแสดงผลของเว็บเบราว์เซอร์ เพื่อระบุพิกัดหรือสั่งคลิกที่รูปภาพดังกล่าวโดยตรง (Image-based Automation) โดยเชื่อมต่อจากบล็อก **Start (Trigger)** ไปยังบล็อก **Open URL** และส่งต่อข้อมูลไปยังบล็อก **Image Search**

## Workflow Blocks

| Block | Type | Description |
|-------|------|-------------|
| **Start (Trigger)** | `BlockBasic` | จุดเริ่มต้นการทำงาน พร้อมระบบ Parameters สำหรับสร้าง UI Form รับข้อมูลผู้ใช้ |
| **Open URL** | `BlockBasicWithFallback` | เปิดหน้าเว็บเป้าหมาย (ในตัวอย่างใช้ `facebook.com`) |
| **Image Search** | `BlockBasicWithFallback` | ค้นหารูปภาพเป้าหมายบนหน้าจอตามค่าพิกัดและความถูกต้องที่กำหนด พร้อมรองรับการคลิกภาพ |

## การตั้งค่าบล็อก Image Search

บล็อก **Image Search** ช่วยแปลงการควบคุมแบบเดิม ๆ จากการใช้ Selector (XPath / CSS) มาเป็นการตรวจหาด้วยวิชันคอมพิวเตอร์ (Computer Vision) เพื่อระบุตำแหน่งของวัตถุหรือปุ่มบนเว็บจากภาพต้นแบบ

### ค่าพารามิเตอร์

| พารามิเตอร์ | คำอธิบาย | ตัวเลือก / รายละเอียดเพิ่มเติม |
| :--- | :--- | :--- |
| **Mode Dropdown** | รูปแบบการระบุรูปภาพที่จะใช้ค้นหา | `Path` (เส้นทางที่อยู่ไฟล์ภาพ เช่น `C:/images/btn.png`) <br/> `Base64` (ข้อมูลรูปภาพแบบเข้ารหัสข้อความ Base64) |
| **Image Path Input** | ฟิลด์สำหรับกรอกเส้นทางไฟล์ภาพหรือโค้ด Base64 | มีปุ่มสลักรูปภาพ 🖼️ เพื่อเลือกรูปจากเครื่อง และปุ่มสอดแทรกตัวแปรส้ม `{x}` |
| **Algorithm (Algo)** | อัลกอริทึมที่ใช้ค้นหารูปภาพเป้าหมาย | `Tpl` (Template Matching สำหรับการค้นหาภาพที่ขนาดเท่ากันเป๊ะ) <br/> `Kaze` (สำหรับค้นหาภาพที่มีความยืดหยุ่นในเรื่องขนาดและมุมหมุน) |
| **Take a screenshot of** | ขอบเขตพื้นที่หน้าจอที่จะแคปเจอร์มาวิเคราะห์ภาพ | `A page` (ทั้งหน้าเว็บ) <br/> `An element` (เฉพาะกรอบอิลิเมนต์ที่เจาะจง) |
| **Click Image** | ติ๊กเลือกหากต้องการให้ระบบทำการคลิกเมาส์ซ้ายที่กึ่งกลางของภาพเมื่อค้นหาพบ | เปิด/ปิด (Boolean) |
| **RGB Enable** | ติ๊กเลือกเพื่อให้เปรียบเทียบค่าสีจริงของรูปภาพ (Red, Green, Blue) | ช่วยเพิ่มความแม่นยำ ป้องกันการสับสนของเฉดสีใกล้เคียง |
| **Threshold** | ค่าระดับความไว/ความคล้ายคลึงในการตรวจจับภาพ (0.0 ถึง 1.0) | เช่น `0.1` ยิ่งค่าน้อยยิ่งต้องการความเหมือนสูงมาก |
| **Coordinates Output Variables** | ชื่อตัวแปรที่จะใช้บันทึกพิกัด X, Y ที่ค้นหาพบ | เพื่อใช้นำค่าพิกัดไปรันคลิกในบล็อกอื่น ๆ ต่อไป |
| **Image / Capture Buttons** | ปุ่มลัดสำหรับอัปโหลดภาพ หรือกดดึงภาพแคปหน้าจอเพื่อทดสอบ | ปุ่มลัดระบบ |
| **Timeout** | ระยะเวลาจำกัดในการพยายามค้นหาภาพก่อนรายงานข้อผิดพลาด | หน่วยเป็นมิลลิวินาที (เช่น `5000` ms) |
| **Delay time** | เวลาหน่วงก่อนจะเริ่มค้นหาภาพ (มิลลิวินาที) | หน่วยเป็นมิลลิวินาที (เช่น `1000` ms) |

### กรณีการนำไปใช้งาน (Use Cases)

*   **คลิกปุ่มที่ไม่มี Selector แน่นอน**: สำหรับหน้าเว็บบางหน้าที่มีโครงสร้างโค้ดเปลี่ยนแปลงตลอดเวลา หรือเป็น Canvas / Flash ที่หา XPath ไม่ได้ สามารถใช้การจำรูปปุ่มเพื่อสั่งคลิกได้
*   **ตรวจสอบสถานะหน้าเว็บ**: ใช้ค้นหาไอคอน เช่น ไอคอนเครื่องหมายถูกสีเขียว หรือไอคอนโหลดค้าง เพื่อเช็คว่าการทำงานสำเร็จแล้วหรือยัง
*   **ตรวจจับรูปภาพ CAPTCHA / การแจ้งเตือน**: ค้นหาภาพสัญลักษณ์แจ้งเตือนแบบป็อปอัปที่ขึ้นมาขัดจังหวะ เพื่อสั่งปิดอัตโนมัติ

---

### ลำดับการทำงานของ Workflow นี้

1.  **Start (Trigger)** — เริ่มต้นชุดคำสั่ง Automation
2.  **Open URL** — เปิดเบราว์เซอร์เข้าสู่ระบบเป้าหมาย `https://www.facebook.com/`
3.  **Image Search** — เริ่มประมวลผลจับภาพหน้าจอและเปรียบเทียบหาภาพเป้าหมายตามอัลกอริทึมที่เลือก หากเจอก็จะสั่งคลิกหรือส่งคืนค่าพิกัด
